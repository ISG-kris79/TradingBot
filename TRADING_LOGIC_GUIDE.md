# 트레이딩봇 진입/익절/청산 로직 가이드

## 📋 개요

현재 구현된 트레이딩봇은 **롱(LONG)과 숏(SHORT) 포지션을 완전히 지원**하며, 다층적인 AI 기반 필터링과 스마트 트레일링 스탑 시스템을 갖추고 있습니다.

---

## 📊 **1. 진입 로직** (TradingEngine.cs)

### 진입 신호 판별
- `decision == "LONG"` 또는 `decision == "SHORT"`로 구분
- 여러 전략에서 시그널 수신: Transformer, ElliottWave, Major 코인 전략 등

### 진입 전 필터링 (롱/숏 동일 적용)

#### 1) 워밍업 체크
- 봇 시작 후 일정 시간 진입 금지
- `IsEntryWarmupActive()` 메서드로 검증

#### 2) 서킷 브레이커
- 리스크 매니저가 발동된 상태에서 자동 매매 차단
- `_riskManager.IsTripped` 확인

#### 3) ATR 변동성 필터
- 5분봉 기준 ATR 비율 계산
- ATR 비율이 **2.0x 이상**: 진입 금지 (흔들기 구간)
- ATR 비율이 **1.5x 이상**: 변동성 상승 경고

#### 4) AI 필터 (예측값 기반)
- **롱**: 상승 확률 45~60% 이상 필요 (전략별 차등)
  - MAJOR 전략: 45% (자체 스코어링 보유)
  - 기타 전략: 60%
  
- **숏**: 더 엄격한 필터링 적용
  - 하락 확률 60% 이상 필요
  - RSI 35 초과 필요 (과매도 회피)

#### 5) RL(강화학습) 필터
- PPO 에이전트의 액션 값 확인
- **롱 진입**: 에이전트가 Sell(2) 액션 추천 시 보류
- **숏 진입**: 에이전트가 Buy(1) 액션 추천 시 보류

#### 6) 손익비 검증 (ElliottWave 기반)
- `Risk:Reward >= 1:2` 조건 검증
- Risk: 진입가 - 1파 저점
- Reward: TP 가격 - 진입가
- 손익비 부족 시 진입 차단

#### 7) 슬리피지 검증
- 호가창 최우선 호가 확인
- **롱**: `BestAsk > CurrentPrice × 1.0005` 시 진입 취소 (+0.05%)
- **숏**: `BestBid < CurrentPrice × 0.9995` 시 진입 취소 (-0.05%)

### 주문 실행
- 지정가 주문으로 슬리피지 방어
- **롱**: `BUY` 주문 (최우선 매도호가 기준)
- **숏**: `SELL` 주문 (최우선 매수호가 기준)
- 수량 계산: `(증거금 × 레버리지) / 현재가`
- 거래소 규격(StepSize) 맞춤 조정

### 진입 후 처리
- 포지션 정보 저장: `_activePositions[symbol]`
- `MonitorPositionStandard()` 호출로 감시 리프 시작
- ElliottWave 3파동 정보 및 피보나치 레벨 기록

## 💰 **2. 익절 로직 (포함: 트레일링 스탑)**

### 2-1. 3단계 스마트 방어 시스템 (PositionMonitorService.cs)

#### **1단계: ROE 10% 도달 → 본절 보호(Break-even)**

**롱 포지션:**
```
진입가 + 0.1% = 본절 보호 스탑
```

**숏 포지션:**
```
진입가 - 0.1% = 본절 보호 스탑
```

- 수수료 포함 본절 확보
- ROE 10% 달성하면 자동 활성화
- 진입가 마이너스 손실 방지

#### **2단계: ROE 15% 도달 → 수익 잠금 (최소 ROE 7%)**

**롱 포지션:**
```
진입가 + 0.35% = 수익 잠금 스탑
```

**숏 포지션:**
```
진입가 - 0.35% = 수익 잠금 스탑
```

- 2단계 도달 시 자동 활성화
- 최소 ROE 7% 확보

#### **3단계: ROE 20% 도달 → 타이트 트레일링 (최소 ROE 18%)**

**롱 포지션:**
```
스탑 = 최고가 - 0.15%  (간격은 0.15% = 0.0015)
최소 스탑 = 진입가 + 18% ÷ 레버리지%  (절대 아래로 내려가지 않음)
```

**숏 포지션:**
```
스탑 = 최저가 + 0.15%
최소 스탑 = 진입가 - 18% ÷ 레버리지%
```

특징:
- 현재가가 상승(롱)/하락(숏) 시에만 스탑 업데이트
- **One-Way Trailing**: 절대 역방향으로 이동하지 않음
- ROE 18% 아래로 절대 내려가지 않음 (최소 수익 보호)

### 2-2. 하이브리드 AI 기반 익절 (HybridExitManager.cs)

#### **1차 익절: AI 예측 가격 도달 → 50% 부분 익절**

**롱 포지션:**
```
현재가 >= PredictedPrice → 50% 익절
```

**숏 포지션:**
```
현재가 <= PredictedPrice → 50% 익절
```

- Transformer 모델의 PredictedPrice 기준
- AI가 목표 도달 시 자동 트리거
- 잔량 50%는 추가 익절 신호 대기

#### **2차 익절: 지표 기반 전량 익절**

**롱 포지션:**
- RSI >= 80: 과매수 극한 → 전량 익절
- BB 상단 강하게 이탈 후 다시 내려옴 → 전량 익절

**숏 포지션:**
- RSI <= 20: 과매도 극한 → 전량 익절
- BB 하단 이탈 후 반등 → 전량 익절

### 2-3. ATR 동적 트레일링 스탑 (HybridExitManager.cs)

#### 작동 원리
- ROE와 RSI 조합으로 스탑 간격 동적 조정
- ATR(Average True Range) 기반 변동성 반영

#### 간격 계산 (ATR 멀티플라이어)

| ROE 상태 | RSI 상태 | 멀티플라이어 | 설명 |
|---------|---------|-----------|------|
| < 10% | 모두 | × 1.5 | 진입 직후, 휩소 방어 |
| 10~20% | < 70 | × 1.0 | 추세 진행 중 |
| > 10% | 70~80 | × 0.5 | 고점 근접 (과열) |
| > 10% | 80+ | × 0.2 | 극단적 과열 (피날레) |

#### 롱/숏 스탑 가격 계산

**롱:**
```
트레일링 거리 = ATR × 멀티플라이어
스탑 = 최고가 - 트레일링 거리

[안전장치] ROE >= 15% 시:
  스탑 < 진입가 → 스탑 = 진입가 (본절 보호)
```

**숏:**
```
트레일링 거리 = ATR × 멀티플라이어
스탑 = 최저가 + 트레일링 거리

[안전장치] ROE >= 15% 시:
  스탑 > 진입가 → 스탑 = 진입가
```

### 2-4. SIDEWAYS 모드 (횡보장 전용)

#### 활성화 조건
- `mode == "SIDEWAYS"` (Transformer 전략)
- 또는 `TakeProfit > 0 && StopLoss > 0` (커스텀 값 설정)

#### 익절 로직

**롱 포지션:**
```
현재가 >= customTakeProfitPrice → 50% 익절
→ 잔여 50% 진입가 보호 (본절가 스탑)
최종: 현재가 <= 진입가 → 전량 청산
```

**숏 포지션:**
```
현재가 <= customTakeProfitPrice → 50% 익절
→ 잔여 50% 진입가 보호
최종: 현재가 >= 진입가 → 전량 청산
```

## 🚨 **3. 청산 로직**

### 청산 조건

#### 1) 방어적 스탑 터치 (3단계 시스템)
**롱:**
```
currentPrice <= protectiveStopPrice → 즉시 청산
```

**숏:**
```
currentPrice >= protectiveStopPrice → 즉시 청산
```

#### 2) 메이저 익절
```
currentROE >= TargetRoe (기본값: 50%)
```

#### 3) 메이저 손절
```
currentROE <= -StopLossRoe (기본값: -20%)
```

#### 4) 지표 기반 즉시 익절
- BB 회귀 신호
- 다중 지표 동시 신호 (3개 이상 발생)
- `AdvancedExitCalculator` 판정

#### 5) 절대 손절
```
ROE <= -20% (HybridExitManager)
```

#### 6) AI 예측 반전
```
롱: PredictedPrice < 진입가 → 즉시 청산
숏: PredictedPrice > 진입가 → 즉시 청산
```

### 청산 주문 실행

**롱 청산:**
```
SELL 주문 (reduceOnly=true)
수량 = abs(포지션 수량)
```

**숏 청산:**
```
BUY 주문 (reduceOnly=true)
수량 = abs(포지션 수량)
```

### 청산 후 검증 (엄격 모드)

1. 주문 성공 후 거래소 포지션 재조회
2. 포지션 수량 0 확인
3. **반대 방향 포지션 감지 시 즉시 재청산**
   - 예: 롱 청산 후 숏이 생성되면 긴급 재청산
4. 포지션 데이터 정리 (`CleanupPositionData()`)

### 안전장치: 서버사이드 손절 주문

진입 직후 Stop-Market 주문 설정:

**롱:**
```
Stop Price = 진입가 × (1 - StopLossROE / 레버리지 / 100)
```

**숏:**
```
Stop Price = 진입가 × (1 + StopLossROE / 레버리지 / 100)
```

## ✅ **4. 롱/숏 완벽 지원 확인**

### 코드 패턴 일관성

모든 로직에서 `isLong` 변수로 분기 처리:

```csharp
// 패턴 1: 조건 분기
if (isLong)
    로직A ("위쪽 방향")
else
    로직B ("아래쪽 방향")

// 패턴 2: 가격 비교
bool condition = isLong 
    ? currentPrice >= targetPrice
    : currentPrice <= targetPrice;

// 패턴 3: 스탑 계산
decimal stop = isLong
    ? entryPrice - distance
    : entryPrice + distance;
```

### 지원 범위

| 기능 | 롱 | 숏 |
|-----|---|---|
| 진입 신호 | ✅ LONG | ✅ SHORT |
| AI 필터 | ✅ | ✅ (더 엄격) |
| 변동성 필터 | ✅ | ✅ |
| 손익비 검증 | ✅ | ✅ |
| 슬리피지 방어 | ✅ | ✅ |
| 3단계 트레일링 | ✅ | ✅ |
| ATR 동적 스탑 | ✅ | ✅ |
| 지표 기반 익절 | ✅ (RSI 80) | ✅ (RSI 20) |
| SIDEWAYS 모드 | ✅ | ✅ |
| 서버 손절 | ✅ | ✅ |
| 포지션 청산 | ✅ SELL | ✅ BUY |
| 반대방향 감지 | ✅ | ✅ |

## 🔧 **5. 설정값 참고**

### 기본 설정 (appsettings.json)

| 항목 | 기본값 | 설명 |
|-----|-------|------|
| DefaultLeverage | 20 | 레버리지 배수 |
| DefaultMargin | 200 | 진입당 증거금 (USDT) |
| TargetRoe | 50% | 메이저 익절 ROE |
| StopLossRoe | 20% | 메이저 손절 ROE |
| BreakEvenROE | 10% | 1단계 본절 보호 |
| ProfitLockROE | 15% | 2단계 수익 잠금 |
| TightTrailingROE | 20% | 3단계 타이트 트레일링 |

### ROE 계산 공식

```
ROE% = (가격변동률 × 레버리지) × 100

예시 (20배 레버리지):
- 1% 상승 → ROE = +20%
- 1% 하락 → ROE = -20%
```

## 📈 **6. 트레이드 플로우 예시**

### 롱 진입 → 익절 시나리오

```
1. [진입]
   - Decision: LONG
   - 진입가: $100
   - 수량: 400 (20배 레버리지)

2. [1단계: ROE 10% 도달]
   - 가격: $101 (ROE = +20%)
   - 스탑: $100.10 (진입가 + 0.1%)
   - 상태: "본절 보호 활성화"

3. [2단계: ROE 15% 도달]
   - 가격: $101.5 (ROE = +30%)
   - 스탑: $100.35 (진입가 + 0.35%)
   - 상태: "수익 잠금 활성화"

4. [3단계: ROE 20% 도달]
   - 가격: $102 (ROE = +40%)
   - 스탑: $102 × (1 - 0.0015) = $101.85
   - 상태: "타이트 트레일링 활성화"

5. [청산]
   - 가격: $101.80 (스탑보다 낮음)
   - 청산 주문: SELL 400
   - 최종 ROE: +36%
   - 상태: 포지션 종료
```

### 숏 진입 → 익절 시나리오

```
1. [진입]
   - Decision: SHORT
   - 진입가: $100
   - 수량: 400 (20배 레버리지)

2. [1단계: ROE 10% 도달]
   - 가격: $99 (ROE = +20%)
   - 스탑: $99.90 (진입가 - 0.1%)
   - 상태: "본절 보호 활성화"

3. [2단계: ROE 15% 도달]
   - 가격: $98.5 (ROE = +30%)
   - 스탑: $99.65 (진입가 - 0.35%)
   - 상태: "수익 잠금 활성화"

4. [3단계: ROE 20% 도달]
   - 가격: $98 (ROE = +40%)
   - 스탑: $98 × (1 + 0.0015) = $98.15
   - 상태: "타이트 트레일링 활성화"

5. [청산]
   - 가격: $98.20 (스탑보다 높음)
   - 청산 주문: BUY 400
   - 최종 ROE: +36%
   - 상태: 포지션 종료
```

## 🎯 **결론**

이 트레이딩봇은 **고도화된 다층 방어 시스템**으로 설계되었으며:

✅ **롱/숏 완전 대칭 지원**
