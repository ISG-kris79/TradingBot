# Release v2.2.1 - 청산 알림 개선 및 PnL 계산 정확도 향상

## 🎯 주요 개선사항

### 📱 텔레그램 청산 알림 메시지 개선
- ✅ 메시지 언어를 영어에서 **한글로 변경**
- ✅ **금일 누적 손익(총수익금)** 정보 추가
- ✅ 익절/손절 구분 이모지 자동 선택 (💰 익절 / 📉 손절)
- ✅ 타임스탬프 형식 개선 (yyyy-MM-dd HH:mm:ss)
- ✅ Push 알림 제목도 한글로 변경

### 💰 PnL 계산 정확도 개선
- ✅ **거래 수수료(왕복 0.08%) 반영**
- ✅ **슬리피지(0.05%) 차감**
- ✅ 모든 청산 경로에 적용 (전량청산, 부분청산, 외부청산, 동기화청산)
- ✅ 디버깅용 상세 로그 추가: 순수PnL / 수수료 / 슬리피지 / 최종PnL 출력

**예시**: 순수 손실 -9.05 USDT → 실제 손실 -12.96 USDT (수수료 0.16 + 슬리피지 3.75)

### 🔄 거래소 포지션 자동 동기화 시스템
- ✅ 거래소에는 없지만 로컬에 남아있는 포지션 자동 감지
- ✅ 청산 기록 자동 복구 (`MISSED_CLOSE_SYNC` 전략)
- ✅ WebSocket 연결 끊김/봇 중지 중 청산 자동 보정
- ✅ **30분 주기 자동 동기화 실행**

### 🖱️ 수동 동기화 UI 추가
- ✅ TradeHistory 탭에 **"🔄 Sync" 버튼** 추가
- ✅ 필요시 즉시 동기화 실행 가능
- ✅ 동기화 완료 시 자동 알림 및 자동 갱신

## 🐛 버그 수정

- ✅ 누락된 청산 기록 자동 복구
- ✅ 외부 청산 포지션 TradeHistory 미반영 문제 해결
- ✅ PnL 계산 시 실제 거래 비용 누락 문제 해결

## 🔧 기술적 개선사항

- `TradingEngine._lastPositionSyncTime` 타이머 추가 (30분 주기)
- 동기화 시 `IsCloseInProgress` 상태 확인하여 중복 처리 방지
- 현재가 조회 실패 시 진입가 폴백 처리
- 동기화 완료 시 `OnTradeHistoryUpdated` 이벤트 발생
- 모든 PnL 계산 로직에 수수료 및 슬리피지 반영

## 📝 문서

- `SLIPPAGE_FEE_TRACKING.md` 추가: 슬리피지 및 수수료 추적 가이드

## 🚀 설치 방법

## ⚠️ 주의사항

- 이 버전부터 PnL 계산에 수수료와 슬리피지가 반영되므로, 이전 버전보다 실제 손익이 더 정확하게 표시됩니다.
- 금일 누적 손익은 RiskManager의 `DailyRealizedPnl`을 기반으로 계산됩니다.
