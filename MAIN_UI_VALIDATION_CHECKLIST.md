# Main UI 검증 체크리스트 (정렬 / ROI / RISK)

## 목적
- 메인 마켓 시그널 그리드에서 다음 3가지가 정상 동작하는지 확인한다.
  1. 정렬 우선순위: 메이저 4개 → 진입 코인 → 나머지
  2. ROI 실시간 갱신: 진입 직후 초기화 + 가격 변동마다 갱신
  3. RISK(SL/TP) 표시: SL/TP 값이 컬럼에 즉시 노출

---

## 사전 준비
- 앱 실행: `dotnet build TradingBot.csproj -c Debug` 후 앱 실행
- 거래소 연결 상태 확인 (실거래 또는 모의거래)
- 메인 화면 `마켓 시그널` 그리드가 표시되는 상태

---

## 1) 정렬 검증 (메이저4 → 진입 코인)

### 기대 동작
- 최상단 우선순위는 항상 메이저 4개: `BTCUSDT`, `ETHUSDT`, `SOLUSDT`, `XRPUSDT`
- 그 다음은 현재 진입 중(`IsPositionActive=true`) 코인
- 그 다음 나머지 코인

### 테스트 절차
1. 앱 시작 직후 마켓 시그널 그리드 확인
2. 메이저 4개가 상단 그룹에 있는지 확인
3. 메이저 외 코인 1개 진입(또는 진입 상태 동기화) 유도
4. 메이저 4개 아래에 진입 코인이 우선 배치되는지 확인
5. 진입 종료 후 해당 코인이 다시 하위 그룹으로 내려가는지 확인

### 판정 기준
- [ ] 메이저 4개가 항상 상단 그룹 유지
- [ ] 진입 코인이 메이저 그룹 다음 우선순위로 정렬
- [ ] 상태 변경 시 정렬이 자동 반영

---

## 2) ROI 실시간 갱신 검증

### 기대 동작
- 진입 직후 ROI가 0 부근에서 시작
- 가격이 틱 단위로 변할 때 ROI가 즉시 갱신
- LONG/SHORT 방향에 따라 ROI 부호가 올바르게 변함

### 테스트 절차
1. LONG 포지션 1개 진입
2. 진입 직후 ROI 값 확인 (0 또는 매우 근접)
3. 가격 상승/하락 구간에서 ROI 변화를 5~10틱 관찰
4. SHORT 포지션에서도 동일하게 확인

### 판정 기준
- [ ] 진입 직후 ROI가 0으로 초기화됨
- [ ] 가격 변동마다 ROI가 멈춤 없이 갱신됨
- [ ] LONG: 가격 상승 시 ROI 상승 / SHORT: 가격 하락 시 ROI 상승

---

## 3) RISK(SL/TP) 컬럼 검증

### 기대 동작
- 포지션 활성 시 `RISK (SL/TP)` 컬럼에 즉시 값 표시
- 둘 다 있을 때: `SL: ... | TP: ...`
- 한쪽만 있어도 단일 값(`SL: ...` 또는 `TP: ...`) 표시
- 비활성 포지션은 `-` 표시

### 테스트 절차
1. 포지션 진입 후 `RISK (SL/TP)` 컬럼 확인
2. SL/TP 둘 다 설정된 케이스 확인
3. SL 또는 TP 한쪽만 있는 케이스 확인
4. 포지션 종료 후 `-`로 돌아가는지 확인

### 판정 기준
- [ ] 활성 포지션에 RISK 값이 표시됨
- [ ] 값 변경 시 컬럼이 즉시 갱신됨
- [ ] 포지션 종료 시 `-`로 초기화됨
